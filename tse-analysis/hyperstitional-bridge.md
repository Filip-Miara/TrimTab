## Structural Hypotheses

### H-1: Per-layer normalization unlocks R² ceiling
| Attribute | Value |
|-----------|-------|
| **Type** | Structural |
| **Statement** | Switching from global normalization (mean(dim=(0,1))) to per-layer normalization (mean(dim=(1))) increases R² by ≥0.02 on the BnB validation set, because per-layer normalization preserves layer-specific velocity signal that global normalization destroys. |
| **Falsification** | R² improves < 0.01 after 5 epochs of training with per-layer normalization on the same 90K trajectories. |
| **Confirmation** | R² ≥ 0.868 (current 0.848 + 0.02) after per-layer norm training. |
| **Minimum Experiment** | Re-normalize existing data per-layer. Train for 5 epochs with B1 configuration. Compare to baseline (same config, global norm). |
| **Risk** | Low — reverting to global norm is trivial. |
| **Value** | If true, confirms the #1 master regulator is correct. Opens path to R²=0.90+. |

### H-2: Velocity targets lie on low-dimensional manifold
| Attribute | Value |
|-----------|-------|
| **Type** | Structural |
| **Statement** | The 3584-dimensional velocity vectors at each layer lie on a manifold with intrinsic dimensionality ≤ 200 (PCA 90% variance explained), because velocity is a smooth function of hidden states which are themselves generated by lower-dimensional attention+MLP operations. |
| **Falsification** | PCA requires > 500 dimensions to explain 90% of velocity variance. |
| **Confirmation** | PCA 90% explained variance achieved with ≤ 200 components. |
| **Minimum Experiment** | Flatten velocity targets to [N×28, 3584]. Fit PCA. Compute cumsum of explained variance ratio. |
| **Risk** | None — purely diagnostic. |
| **Value** | If true, justifies PCA compression (14× reduction) which enables more efficient capacity use, denoising, and potentially large R² gains. |

### H-3: Layer-specific gradient imbalance reduces effective capacity
| Attribute | Value |
|-----------|-------|
| **Type** | Structural |
| **Statement** | Early layers (1-10) receive ≥ 70% of the total gradient magnitude during training because their velocities have higher magnitude, causing late layers (20-28) to be under-optimized. This under-optimization limits overall R². |
| **Falsification** | Per-layer gradient norms show < 2× difference between max and min layers, or late-layer R² is not significantly lower than early-layer R². |
| **Confirmation** | Gradient norm ratio (layer 1 / layer 28) ≥ 3×, AND late-layer R² ≥ 0.05 below early-layer R². |
| **Minimum Experiment** | Log per-layer gradient norms during training. Compute per-layer R² after training. |
| **Risk** | Low — monitoring only. |
| **Value** | If true, justifies per-layer loss weighting (inverse variance), which should improve late-layer predictions by 0.03-0.05 R². |

## Relational Hypotheses

### H-4: AWQ→BnB distribution shift is approximately affine
| Attribute | Value |
|-----------|-------|
| **Type** | Relational |
| **Statement** | The difference between AWQ and BnB hidden states can be approximated by a per-feature affine transformation (scale + shift): h_AWQ ≈ a·h_BnB + b, where a and b are 3584-dimensional vectors. This is because AWQ quantization applies deterministic scaling and rounding, which manifests as a linear transformation in each feature. |
| **Falsification** | AWQ→BnB residual after optimal affine alignment has R² < 0.60 (i.e., linear model explains <60% of variance). |
| **Confirmation** | AWQ→BnB residual after affine alignment has R² ≥ 0.80. |
| **Minimum Experiment** | Collect paired BnB/AWQ hidden states on 1000 tokens. Fit per-feature linear regression: h_BnB = w·h_AWQ + b. Measure R². |
| **Risk** | Low — simple regression. |
| **Value** | If true, a simple correction layer (affine transform) can bridge AWQ→BnB, potentially recovering most of the R² drop without any TT changes. |

### H-5: Directional error (not magnitude) is the primary driver of steering quality
| Attribute | Value |
|-----------|-------|
| **Type** | Relational |
| **Statement** | For KV-cache steering, directional accuracy of velocity (cosine similarity with true velocity) correlates with downstream reasoning accuracy (GSM8K score) at r ≥ 0.5, while magnitude accuracy (ratio of predicted/norm) correlates at r ≤ 0.2. This is because steering direction determines whether KV-cache moves toward better reasoning states, while magnitude only determines step size. |
| **Falsification** | (a) Cosine similarity with true velocity has r < 0.2 with GSM8K accuracy, OR (b) magnitude accuracy has r ≥ cosine accuracy. |
| **Confirmation** | Cosine r ≥ 0.5 AND magnitude r ≤ 0.2 with GSM8K accuracy. |
| **Minimum Experiment** | Train 5 TTs with different random seeds → 5 different velocity predictions per trajectory. Apply each to KV-cache steering. Measure GSM8K accuracies. Correlate velocity metrics with accuracies. |
| **Risk** | Medium — requires end-to-end eval infrastructure. |
| **Value** | If true, justifies prioritizing cosine loss > MSE. If false, the entire velocity prediction approach may need rethinking. |

### H-6: The noise ceiling is below current R²
| Attribute | Value |
|-----------|-------|
| **Type** | Relational |
| **Statement** | The noise ceiling (irreducible MSE from stochasticity in Qwen's forward pass) is ≤ 30% of current validation MSE, meaning ≥ 70% of the error is reducible through better architecture/algorithm. This is because Qwen's hidden states are largely deterministic given the input (the model has negligible dropout or sampling-induced variance). |
| **Falsification** | Noise MSE / total MSE ≥ 0.50. |
| **Confirmation** | Noise MSE / total MSE ≤ 0.30. |
| **Minimum Experiment** | Run Qwen forward pass twice on 100 inputs (same config, greedy decoding). Compute per-layer velocity for each run. Compute MSE between run-A and run-B velocities. Denominate by total MSE of predictions. |
| **Risk** | High (existential for project) — confirmed noise ceiling would redirect entire effort. |
| **Value** | If true, removes the primary objection ("R²=0.85 is the ceiling") and justifies full investment in improvement. If false, redirects effort to noise reduction or alternative approaches. |

## Potential Hypotheses

### H-7: Multi-format training creates quantization-robust representations
| Attribute | Value |
|-----------|-------|
| **Type** | Potential |
| **Statement** | Training TT on a mixture of trajectories from BnB + AWQ + GPTQ variants of Qwen (1:1:1 ratio) will produce a TT that achieves AWQ R² ≥ 0.70 without BnB forgetting (BnB R² ≥ 0.80). This emergent capability arises because TT learns to ignore quantization-specific artifacts and focus on shared velocity structure. |
| **Falsification** | After multi-format training, either (a) AWQ R² < 0.65, or (b) BnB R² drops > 0.05 from baseline. |
| **Confirmation** | AWQ R² ≥ 0.70 AND BnB R² ≥ 0.80. |
| **Minimum Experiment** | Generate 30K AWQ trajectories. Combine with 30K BnB trajectories. Train from scratch. Validate on both formats. |
| **Risk** | Medium — data generation cost (~2 days) lost if hypothesis fails. |
| **Value** | If true, solves the #1 known issue (AWQ transfer) AND the emergent property suggests robustness to future quantization formats. |

### H-8: PCA-compressed velocity prediction improves R²
| Attribute | Value |
|-----------|-------|
| **Type** | Potential |
| **Statement** | Compressing velocity targets to top-256 PCA components (from 3584) and predicting in PCA space yields higher R² (measured in original space via reconstruction) because (a) PCA removes noise dimensions, (b) the 48M model capacity is more effectively allocated across 256 dims vs 3584 dims (187K params/dim vs 13K params/dim). Expected gain: +0.02 to +0.08 R². |
| **Falsification** | R² in original space after PCA compression is ≤ baseline R². |
| **Confirmation** | R² ≥ 0.868 (baseline + 0.02) in original space. |
| **Minimum Experiment** | Fit PCA on training velocity targets (K=256). Transform targets to PCA space. Modify TT output to 256 dims. Train. Transform predictions back to original space for evaluation. |
| **Risk** | Low — PCA is well-understood and invertible. |
| **Value** | If true, provides a 14× efficiency gain AND better predictions. Justifies significant refactoring. |

### H-9: Direction+magnitude decomposed loss improves directional accuracy
| Attribute | Value |
|-----------|-------|
| **Type** | Potential |
| **Statement** | Decomposing the loss into cosine similarity (for direction) + Huber loss (for magnitude) with learned gating α increases cosine similarity by ≥ 0.05 compared to MSE-trained TT, because the decomposed loss separately optimizes the two distinct aspects of velocity that matter for steering. |
| **Falsification** | Cosine similarity improves < 0.02 after decomposed loss training. |
| **Confirmation** | Cosine similarity ≥ 0.820 (baseline 0.770 + 0.05). |
| **Minimum Experiment** | Implement decomposed loss. Sweep α in {0.1, 0.3, 0.5, 0.7, 0.9}. Train 5 TT variants (one per α). Measure cosine similarity. |
| **Risk** | Low — loss function change only. |
| **Value** | If true, provides a direct improvement in steering signal quality. Combined with H-5 (directional error drives steering quality), this ROI compounds. |

### H-10: Correction network avoids catastrophic forgetting
| Attribute | Value |
|-----------|-------|
| **Type** | Potential |
| **Statement** | Training a small correction network (MLP, 3 layers, 5M params) that maps AWQ hidden states → BnB-equivalent states, while keeping the original BnB-trained TT frozen, will achieve AWQ R² ≥ 0.70 with ZERO BnB forgetting — because the correction network absorbs the distribution shift without modifying the TT. |
| **Falsification** | After correction network training, either (a) AWQ R² < 0.60 despite correction, or (b) the correction network degrades BnB performance. |
| **Confirmation** | AWQ R² ≥ 0.70 AND BnB R² within 0.005 of original. |
| **Minimum Experiment** | Collect paired BnB/AWQ trajectories. Train correction MLP with MSE between corrected-AWQ and BnB hidden states. Feed corrected-AWQ states into frozen TT. Measure AWQ R². |
| **Risk** | Medium — correction network may not generalize to all AWQ patterns. |
| **Value** | If true, solves the AWQ transfer problem with zero interference to the original TT. Enables "one TT per base model, adapter per quantization format." |
