# Recursive Self-Assessment — RankAdaptation TSE Analysis

## Analysis Weaknesses

### Structural Weaknesses
1. **Incomplete codebase coverage**: The analysis read ~15 core files but the codebase has 60+ Python files and 200+ entries. Analysis of the complete adapter zoo (DoRA, BoRA, EDoRA, etc.) was omitted — these may contain relevant techniques for steering.
2. **Skim over data**: The actual trajectory data files (83 batch files, 35GB) were not inspected. The analysis relies entirely on metadata from PROJECT_DEBRIEF.md.
3. **No quantitative re-analysis**: The analysis uses reported numbers (R², accuracy, p-values) without re-running any experiments. Statistical claims are accepted at face value.
4. **Missing `FULL_SESSION_RECOVERY.md`**: This referenced document may contain important session-level details not captured in PROJECT_DEBRIEF.md.

### Relational Weaknesses
1. **No cross-referencing with existing literature**: The analysis should have compared findings with published mechanistic interpretability work (e.g., Anthropic's transformer circuits, Othello GPT probes, induction heads). The trim-tab/death-layer pattern may already be documented in the literature.
2. **No domain-expert reviewers**: The control theory, neuroscience, and systems analogies were generated by a single reviewer without domain expertise verification.
3. **λ interpolation analysis is incomplete**: The predicted optimal λ assumes a linear relationship between correct/incorrect manifolds, which may not hold.

### Potential Weaknesses
1. **Phase 4b emergent discovery depth**: Only 7 unconventional recombinations were generated. The TSE recombinative space is much larger (cross-level + domain-transposed + forbidden pairs + self-application). Many potentially fruitful combinations were missed.
2. **Failure mode analysis limited to 4 testable hypotheses**: The hyperstitional bridge should have generated 10+ hypotheses with falsification criteria. Only 7 were produced.
3. **Phase 9 temporal plan assumes linear progression**: The decision tree assumes Phase A → B → C → D, but in reality, insights from Phases C/D could change the interpretation of Phase A results.

## Blind Spots Discovered

1. **Norm-growth hypothesis**: The analysis surfaced this as a possible alternative explanation for the TT's R², but this was only discovered in Lens 5 (abductive) and Phase 8. Earlier phases assumed TT learns meaningful dynamics without questioning what "meaningful" means.
2. **Per-head granularity**: Identified in Lens 7 (metacognitive) as a persistent blind spot but never fully analyzed. The entire analysis operates at layer granularity; head-level effects could overturn many conclusions.
3. **Phase 0 assumption about capability threshold**: The discrete threshold (40% GSM8K) was treated as a valid construct, but Phase 6 (D4) revealed it might be confounded with instruct-tuning. This was only caught in disparity analysis.
4. **Why it was missed**: The TSE lens cascade is comprehensive but sequential — insights from later lenses (abductive, metacognitive, adversarial) don't retroactively restructure earlier phases. The Phase 0 VOID, Phase 1 pyramid, and Phase 2 lens outputs were produced before the norm-growth hypothesis surfaced.
5. **How to catch next time**: Implement a Phase 2.5 "retrospective inconsistency check" where high-confidence findings from lenses 7-10 are matched against Phase 0 assumptions. Any mismatch triggers a Phase 0 revision cycle.

## Proposed Updates to TSE

1. **Add a Phase 2.5 "Retrospective Check"**: After the lens cascade (Phase 2), before Master Regulators (Phase 3), add a check that identifies findings from later lenses that contradict Phase 0 assumptions or Phase 1 atoms. This prevents blind-spot persistence.
2. **Increase Phase 4b minimum recombination count**: The current skill says "generate X recombinations" without a minimum. Propose: generate at minimum 12 recombinations (3 per class) in full mode.
3. **Null hypothesis generator in Phase 8**: Currently Phase 8 suggests null hypotheses ad hoc. Propose: systematically generate null hypotheses for the top-3 findings from Phase 2's convergence check.

## Negative Space

### What was NOT found in this analysis
1. **No evidence that steering is learned via RL**: Despite Open Question 7 mentioning RL-optimized α, no RL implementation exists in the codebase. This is a gap.
2. **No evidence of multi-head contrastive ensembles**: Open Question 8 mentions bagging bootstrap contrastive pairs, but no code implements this.
3. **No per-token analysis**: The analysis determines if a generation is correct/incorrect (aggregate), but never looks at per-token accuracy. The steering might help at 30% of tokens, hurt at 20%, and be neutral at 50%.
4. **No ablation of the TT architecture**: The TT uses 6 layers, 8 heads, 768 d_model. No sensitivity analysis (e.g., 3 layers vs 12 layers, 4 heads vs 16 heads).

### Why they might not have been found
- The codebase focuses on building functional experiments, not systematic ablation studies. Per-token analysis requires per-token labels, which are expensive to produce.
- RL and multi-head ensembles are listed as "next experiments" that weren't reached.

### Do they warrant separate investigation?
- **RL optimization** (Open Question 7): YES — this is the natural endpoint of the steering research. If manual α gives +20pp, RL-optimized per-token α could give +30-40pp.
- **Per-token analysis**: YES — might reveal that steering helps on some token types (numbers, operations) and hurts on others (prepositions, articles), guiding where to apply it.
